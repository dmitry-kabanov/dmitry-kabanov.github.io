JavaScript: objects and arrays

By Dmitry Kabanov

September 1, 2024

Programming is difficult if the only available data types are numbers, booleans, and strings. Very often one needs to pack several different data items together, to build more or less useful programs.

<!–more–>

This post is based on the Chapter 4 of the Eloquent JavaScript book by Marijn Haverbeke.

Arrays

JavaScript provides a way to pack together a sequence of values in the form of an array. In the simplest form, an array can be created using square brackets syntax:

let listOfNumbers = [2, 3, 5, 8, 13];
console.log(listOfNumbers[0]);
console.log(listOfNumbers[3]);

Indexing into the array is also done with square brackets and starts with zero.

Properties

Syntax like this: Math.max or str.length is used to access a property of some object. Here, max is a function of module Math and length is a property of the str object (we assume it is a string).

There is an alternative syntax to access properties: obj[key], in which the variable key is first evaluated to its value, and then the property of the obj is accessed that corresponds to this value. Note that the key will be casted to a string.

Arrays, as strings, have the length property.

Methods

Properties of the object that hold functions, are also called methods. For example, all strings have the toUpperCase property:

let s = "Hello World!";
console.log(s.toUpperCase());
console.log(typeof s.toUpperCase);

Arrays have two operations push and pop that are used to add a value to the end of the array and to remove a value from the end of the array:

let arr = [1, 2, 3, 4];
console.log("Initial: ", arr);
arr.push(5);
console.log("Push:    ", arr);
arr.pop();
console.log("Pop:     ", arr);

This operations and their names tell us that JavaScript arrays can be used as a stack data structure.

Objects

Objects pack together values of different types with arbitrary properties names (not just integers, like for arrays). The easiest way to create an object is using curly-braces syntax:

let day1 = {
    squirrel: false,
    events: ["work", "touched tree", "pizza"],
};

console.log("Turned to squirrel: ", day1.squirrel);
console.log("Activities: ", day1.events);
console.log("Turned to wolf: ", day1.wolf);

Note that as squirrel and events can be used as variable names, we do not have to enclose them in quotes. If we want to add an object property with, say, spaces, then we have to enclose the property name in quotes and use the square brackets to access the value of the property:

let person = {
    age: 27,
    "child memories": ["first ice cream", "parents", "bike"],
}
console.log("Age: ", person.age);
console.log("Childhood memories: ", person["child memories"]);

Note that in the above example with the object day1, we tried to access a property wolf, which was not defined at object creation, hence when accessing it, we have got undefined.

We can create new properties on objects in JavaScript on the fly:

let o = {
    prop1: "prop1 value",
};
console.log("prop1: ", o.prop1);
console.log("prop2: ", o.prop2);
o.prop2 = "prop2 value";
console.log("prop2: ", o.prop2);

We can check if an object has a property using in operator: propname in obj, which returns true if the property exists.

To see all available property names, use keys() method on any object.

Arrays are specialized objects: typeof [] returns object.

Mutability

Objects properties are mutable, that is, then can be changed from their initial values, in contrast with immutable objects such as numbers and strings.

Objects also have identity, which means that two objects with exactly the same values of properties are not necessarily the same object:

let o1 = { p: 1 };
let o2 = o1;
let o3 = { p: 1 };
console.log("o1 == o2 ? ", o1 == o2);
console.log("o1 == o3 ? ", o1 == o3);

o1.p = 5;
console.log("o1.p = ", o1.p);
console.log("o2.p = ", o2.p);
console.log("o3.p = ", o3.p);

As can be seen from the output, objects o1 and o2 are actually the same object (proper way of saying it is that o1 and o2 are references to the same object), while the object o3 is completely different, although initially o1 and o3 have the same property values:

Note that you can still declare an object with the const keyword, which means that the object can be changed (mutated), however, the variable cannot be pointed to another object:

const o = { p1: 1, p2: 2 };
o.p1 = 10;  // allowed

// not allowed - results in the error "Assignment to constant variable."
o = { p3: 3 };

When objects are checked for equality with the == operator, it returns true only if they are of the same identity. There is no object that does “deep” comparison for property values.

let x = 1;
console.log(`x = ${x}`);
console.log("Hello World!");
// window.location = "https://musicbydima.com";

let x = 1;
console.log(`x = ${x}`);
console.log("Hello World!");
// window.location = "https://musicbydima.com";
console.log(`x again = ${x}`);
